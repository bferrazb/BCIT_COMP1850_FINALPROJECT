FINAL PROJECT - EXPLANATION

Responsive Design

Responsive design was implemented through the use of CSS Grid + Media Queries. The breakpoint occurs at 1500px and disables all the portions pertaining to the desktop version, while activating the mobile version, and vice versa. Embedded CSS was used to better retain control of all the media queries such as to not conflict with the CSS Base file. The changes are: Logo positioning, Navigation bar style, and content positioning. Also, the mobile version has a more simplified design, not containing the same amount of information that can be found in the sidebar of the desktop version.

Javascript & Interactivity Requirements

Javascript can be found in the buttons on the Programs & Account pages. Programs page gives you an alert saying that the item has been added to the cart, once the user clicks on Buy. Account pages has 2 buttons, each of them have a function to redirect to a specific page once clicked on (Home & Programs pages). Other javascript implemented was the live chat button, which once clicked gives the user an alert stating that the feature is still in development. As for interactivity, the accordion tabs inside the account page have a transition on them once the user clicks to show more. There's also a CSS portion which changes the icon from + to - once opened.

Form Requirements

Form is linked to a php file that was re-used from Week 10. It also has a javascript function that creates a fully functional captcha box. The other 2 options that the user has to contact the studio are the live chat and the contact information which can be found on the sidebar and at the bottom of the contact us page.
